Getting Started

System requirements

  • Windows 10/11 (64-bit)
  • 4 GB RAM minimum
  • Internet connection for broker/API access and licence checks

Download and install

  1. Go to the Download page.
  2. Download the latest release ZIP.
  3. Extract to a local folder.
  4. Run Xaltrion.exe.

If SmartScreen appears: click More info, then Run anyway. This is expected for unsigned desktop builds in beta.

Activate your licence key

  1. Open the app and go to Activation if not prompted.
  2. Enter your XALT-XXXX-XXXX-XXXX-XXXX key.
  3. Confirm status is ACTIVE.

Connecting Brokers

Alpaca (paper trading first)

  1. Create an Alpaca account and open a paper environment.
  2. Generate API key + secret.
  3. Use paper endpoint while testing.

Luno (ZAR pairs)

  1. Create API credentials in your Luno account.
  2. Enable required trade permissions.
  3. Use ZAR pairs for local-market strategies.

Binance

  1. Create API key and secret.
  2. Prefer IP allowlisting where possible.
  3. Keep withdrawals disabled for safety.

IG South Africa

  1. Create or use an IG demo account first.
  2. Generate IG API key and login credentials.
  3. Test connectivity from Settings before live mode.

Entering keys in Xaltrion

Open Settings, select the broker tab, add credentials, and run each tab's connection test before enabling live trading.

Understanding the Dashboard

  • Status strip: live state, scan cadence, and most recent agent activity.
  • Positions tile: current holdings, value, and open P&L.
  • Journal: every Claude decision and why it was made.
  • Market events: event feed to contextualize volatility windows.
  • AI spend: Claude/AI usage costs and budget awareness.
  • Cash Out: emergency position liquidate action.

Trading Modes

Paper trading (default)

Paper mode uses simulated balances and order flow. Use this until your setup and guardrails are proven.

Live trading

Switch only after successful paper cycles and validated broker connectivity.

Circuit breaker

Automatic safety stop if daily drawdown crosses threshold. Reset only after you understand the trigger reason.

Settings

  • Perplexity model selection: pick the model balancing speed and cost.
  • Watchlist management: curate symbols and priorities.
  • Broker recommendation settings: tune routing behavior per broker availability.
  • PIN protection: lock down sensitive portal controls.

Troubleshooting

Licence shows "Expires soon" after updating expiry

After extending your subscription expiry in the admin portal, the portal may still show "Expires soon". The JWT token is cached locally with the old expiry date. Fix: Stop Xaltrion. Delete state\license_token.json. Restart - it fetches a fresh token from the server. Note: restarting alone is not enough, the file must be deleted.

Activation rejected - machine already bound

If you reinstall or run on a different machine, activation may fail with a machine binding error. Each licence key is bound to one machine at a time. Fix: Contact support@xaltrion.ai with your licence key. We will clear the machine binding. Then re-enter your key.

Portal won't start - port already in use

A previous instance may still be running. Fix: Open Task Manager, find XaltrionAI.exe or python.exe, end those processes, wait 5 seconds, restart Xaltrion. If it persists, restart your computer.

Windows shows a security warning on first run

Windows SmartScreen may show "Windows protected your PC" on first run. This is expected for software without a code signing certificate. Fix: Click "More info" then "Run anyway". Only appears on first run or after updates. Code signing will be added in a future release.

Licence key not accepted

Check: correct XALT-XXXX-XXXX-XXXX-XXXX format, internet connection active, key not already bound to another machine, check spam folder for key email. If none apply, contact support@xaltrion.ai with your order reference.

Journal shows no research context

If journal entries show no research, the Perplexity integration may need attention. Fix: Open Settings -> Research Level. Verify Perplexity API key is entered. Current recommended model: sonar. Check perplexity.ai dashboard to confirm API plan status.

Light/dark mode resets after page reload

Browser may be blocking localStorage. Fix: Ensure browser allows localStorage for localhost. Do not use private/incognito mode - localStorage clears on close in private mode.
